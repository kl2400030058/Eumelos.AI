# Eumelos — Prototype Demonstration & Experience Guide

This document outlines the core aspects demonstrated by the Eumelos University Portal prototype, specifically focusing on the application flow, user interface, navigation, and functional workflows.

---

### 1. 🔄 Application Flow
The prototype demonstrates a structured, guidance-first entry sequence designed to onboard students smoothly:
*   **Stage 1: Rules & Regulations Acceptance** – The gateway stage requiring agreement to the digital Student Handbook.
*   **Stage 2: Portal Preparing (Loading)** – A 6-second synchronized boot sequence that prepares the environment while providing "Student Tips of the Day."
*   **Stage 3: Public Landing Page** – Immediate entry to the university's main home page after portal initialization.
*   **Stage 4: Personalization Gateway** – A dedicated style selector appearing when entering the private portal area, allowing users to choose their accent colors and theme modes (Light/Dark).
*   **Stage 5: Role Selection & Authentication** – Choosing between Student or Admin roles and entering the secure login environment.
*   **Stage 6: Private Dashboard** – Landing on the persona-specific command center.

---

### 2. 🎨 UI Screens
The prototype highlights a "Glass Black" design system with multiple high-fidelity screens:
*   **Landing Page** – A modern, hero-focused landing page with transparent navigation and feature cards.
*   **Theme Selection Overlay** – A fully interactive personalization booth with real-time accent color previews.
*   **Student Dashboard** – A grid-based overview featuring "My Current GPA," "Overall Attendance," and "Course Progress" cards.
*   **My Schedule (Temporal Grid)** – A weekly visualization showing class timings and active session highlighting.
*   **Course Discovery** – A card-based registry for browsing and enrolling in academic modules.
*   **Admin Command Center** – A data-rich interface for student management, document verification, and system monitoring.
*   **Loading Gateway** – A branded progress screen with animated status updates and rotating tips.

---

### 3. 🧭 Navigation
The prototype features a robust, dual-mode navigation system:
*   **Horizontal Navbar (Public)** – A minimal, top-aligned navbar for the landing and login pages, featuring a "Logo Trigger" to reveal extended links.
*   **Interactive Sidebar (Private)** – A professional sidebar system for the student and admin dashboards that manages core module switching (Dashboard, Schedule, Courses, etc.).
*   **Route Protection** – Intelligent redirection that manages access based on user role and initialization state (e.g., preventing dashboard access before theme selection).
*   **Quick Actions** – A centralized button for frequent tasks like "Send Message" or "Registration."
*   **Metrics Footer** – A persistent footer for students that displays real-time graduation progress and standing.

---

### 4. ⚙️ Functional Workflow
Key workflows demonstrated in the prototype include:
*   **User Onboarding Workflow** – The seamless transition from accepting rules to initializing the interface and choosing a visual theme.
*   **Authentication & Role Switching** – The separate login paths for Students and Admins, leading to customized data views.
*   **Theme Personalization Workflow** – The ability to globally update the system's "Accent Color" and "Mode" (Dark/Light) with instant persistence.
*   **Academic Tracking Workflow** – Students viewing their schedule, monitoring gpa/attendance, and checking semester registration statuses.
*   **Administrative Verification Workflow** – Admins reviewing student registries, verification requests, and auditing security logs.
